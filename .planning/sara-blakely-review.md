# Sara Blakely Gut-Check: GitHub Intake MVP

## Would a real customer pay for this?

**Not yet.** "File an issue, walk away, it ships" sounds sexy — but who's the customer? This plan is written for *you*, not for them. You're solving a workflow problem without proof anyone else has it.

When I sold Spanx, I led with "no panty lines" — not the yarn blend. Where's *your* panty line moment?

## What's confusing? What would make someone bounce?

- **p0/p1 labels** — What are those? Why would I use them? You're assuming I already think like you.
- **PRD** — Normal humans don't know this word. They don't care.
- **Silent automation** — I file an issue and then... nothing? No comment, no notification, no proof it worked? I'd assume it was broken.
- **113 lines of code, no 5-second demo** — Show me it working or I'm out.

## 30-Second Elevator Pitch

"Label a GitHub issue 'p0' — our daemon turns it into a spec, queues it for your AI agent, and tracks everything. You never copy-paste from GitHub again. One label. That's it."

## What would you test first with $0 budget?

Tweet the pitch. Post in one Discord. DM 5 dev friends and ask: "How do you go from GitHub issue to actual work?" If nobody leans in, the problem isn't real enough. Don't write 113 lines to find out you were wrong.

## What's the retention hook?

**There isn't one.** Silent background magic = forgettable. Add a weekly digest: "Intake converted 4 issues this week. Here's what shipped." Make them *feel* the value.

---

**Bottom line:** The tech is clean. The *story* is missing. Sell to a person, not a system.
