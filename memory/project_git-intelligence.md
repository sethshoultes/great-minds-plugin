## What Worked

### Process Patterns to Repeat
- **Dialectic debate** — Steve vs Elon produced genuine synthesis, not compromise
- **Multi-lens review** — Maya (copy), Jony (design), Board (strategy) each caught unique issues
- **Scope discipline** — 6+ features cut to achieve <100 line core
- **Essence document** — anchored all decisions and prevented drift
- **Board conditions** — mandatory requirements ensured baseline quality

### Technical Approaches That Were Effective
- **Parallel git execution** — Promise.all() for independent commands
- **Zero configuration** — "ships opinions, not options"
- **Mentor voice** — "Tread carefully" instead of "WARNING: DANGER"
- **Named constants** — extracted magic numbers to config object
- **Risk levels not scores** — LOW/MEDIUM/HIGH vs numeric values

### Tools That Worked Well
- **Rick Rubin essence extraction** — immediately usable creative direction
- **Shonda Rhimes retention roadmap** — converted critique into actionable v1.1 plan
- **Marcus Aurelius retrospective** — honest process evaluation

---

## What Didn't Work

### Process Patterns to Avoid
- **Late strategic review** — Board questions about moat/revenue came after creative investment
- **"TBD" as owner** — deferrals without accountability become fantasies
- **Post-hoc creative review** — Maya and Jony saw finished artifacts, not outlines
- **Score averaging without reconciliation** — 4/10 and 7.5/10 indicate measurement misalignment

### Technical Approaches That Caused Issues
- **English-only regex** — knowingly shipped exclusionary pattern
- **Console.log outcome tracking** — data dies with terminal session
- **No technical spike** — full process executed before validating feasibility

### Where Time Was Wasted
- **Detailed debate on features that got cut** — shorter spike would have identified scope earlier
- **Board review after full implementation** — strategic gaps discovered too late to address

## Refined Frameworks

### Before Round 1 (New Checklist)
- [ ] Revenue model hypothesis documented
- [ ] Competitive moat identified or acknowledged as absent
- [ ] Compounding mechanism defined or explicitly deferred
- [ ] Customer segment specified
- [ ] 2-hour technical spike completed

### For All Deferrals (v2+ items)
Must include:
- **Owner**: Named individual (not "TBD")
- **Deadline**: Calendar date
- **Kill condition**: What causes abandonment
- **Success metric**: How we know it worked

### For Creative Reviews
- Engage at outline stage, not finished artifacts
- Shape direction, don't just polish endpoints

### For Board Review
- Move earlier in pipeline for strategic projects
- Force score reconciliation before verdict
