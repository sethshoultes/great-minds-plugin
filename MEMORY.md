## Key Learnings (Accumulated)

### Process Learnings

1. **Front-load strategic questions** — moat, revenue, compounding before creative investment
2. **Require technical spike** — 2-hour PoC before multi-day deliberation
3. **Involve creative reviewers early** — at outline stage, not finished artifacts
4. **Assign owners to deferrals** — name, date, kill condition for every "v2" item
5. **Reconcile divergent scores** — 3+ point spread needs explicit resolution

### Technical Learnings

1. **<100 lines is achievable** — scope discipline + feature cuts = clean code
2. **Zero config is possible** — "ships opinions, not options"
3. **Parallel execution matters** — Promise.all() for independent git commands
4. **Mentor voice > alarm voice** — "Tread carefully" not "WARNING: DANGER"
5. **Named constants improve maintainability** — extract magic numbers to config objects

### Agency Learnings

1. **Dialectic synthesis works** — genuine opposition produces genuine solutions
2. **Multi-lens review catches what single reviewers miss** — different reviewers see different problems
3. **Board review should come earlier** — strategic questions before creative investment
4. **Invisible value is risky** — hard to price, easy to replicate
5. **v2 fantasies accumulate** — require accountability or acknowledge they may never ship
