# DREAM Cycle: Steve Jobs Brainstorm
**Date:** 2026-04-09
**Focus:** Design Quality, User Delight
**Constraint:** Buildable in one session, clear distribution

---

## Idea 1: **Prose Polish** — AI Writing Coach for WordPress

The Gutenberg editor is functional but *soulless*. Writers paste their draft and get... nothing. No feedback, no delight.

**Prose Polish** lives in the sidebar. As you write, it quietly analyzes your prose. Not grammar — *craft*. Reading level trends. Sentence rhythm visualization (short-long-short). Overused words heat-mapped in the text. A "clarity score" that rewards simplicity.

The magic: it's *encouraging*. Green pulses when you nail a paragraph. Gentle amber when something's muddy. Never red. Never shame. Writers should feel like the tool is their ally, not their editor.

**User:** WordPress bloggers, content marketers, indie writers
**Distribution:** WordPress.org plugin directory (we know this channel)
**Dogfood:** We use it to write our own docs and blog posts
**Tech:** Gutenberg sidebar + Cloudflare Workers AI for analysis
**Session scope:** Single PHP plugin + React sidebar component + one AI endpoint

## Idea 2: **Shortcut Studio** — Visual Keyboard Shortcut Builder

Every power user has keyboard shortcuts they *wish* existed. But creating them requires scripting, Automator, or third-party apps with 47 tabs of settings.

**Shortcut Studio** is a web app with one screen: a visual flow builder. Drag blocks: "When I press ⌘+Shift+P" → "Run this shell command" → "Show notification." Export as a `.shortcut` file that works with native macOS/Windows tooling.

The interface is *joyful*. Keys animate when you press them. Flows have bezier curves. The export button says "Make it real."

**User:** Developers, designers, power users
**Distribution:** Product Hunt launch + dev Twitter/X
**Dogfood:** We build shortcuts for our own agency workflow
**Tech:** Remotion-style React canvas + local file export (no backend needed)
**Session scope:** Static React app, localStorage persistence, file download

## Idea 3: **Commit Theatre** — Turn Git History into Video Changelogs

Release notes are boring. Users don't read them. But everyone watches a 30-second video.

**Commit Theatre** takes a git diff (or GitHub release) and generates a narrated video walkthrough. It shows the files changed (animated file tree), the key code snippets (syntax highlighted, zooming in), and a TTS narrator explaining what changed and why it matters.

Upload your repo URL. Pick a commit range. Get a video. That's it.

**User:** Open source maintainers, dev rel teams, indie hackers
**Distribution:** GitHub Marketplace listing + HN/dev communities
**Dogfood:** We use it for Great Minds Plugin releases
**Tech:** Remotion + Cloudflare Workers AI for narration script + ElevenLabs TTS
**Session scope:** We've already built the Remotion + TTS pipeline — this is a new frontend

---

## Ranking (Steve's Pick): **Commit Theatre**

Why: It's the most *delightful* — turning the mundane (changelogs) into something people actually want to watch. It leverages our existing Remotion expertise. And it's the kind of product that markets itself when people share their videos.

*"Design is not just what it looks like. Design is how it works."* — This works elegantly. Git in, video out.
