## Idea 1: **AgentBench** — Benchmark Your AI Agents in Production

Every team building AI agents faces the same problem: "Is my agent actually getting better?" They tweak prompts, add tools, change models — but measurement is ad-hoc. Screenshots in Slack. Vibes.

**AgentBench** is a lightweight benchmarking service. You define test cases (input scenario → expected behavior). Push your agent through our API. We run it 10x, measure latency, token usage, success rate, and output quality (via LLM-as-judge). Dashboard shows trends over time.

This is picks-and-shovels for the AI gold rush. Every company building agents needs this.

**User:** AI engineering teams, startups building agent products
**Distribution:** HN launch + direct outreach to AI Discord/Slack communities
**Dogfood:** We benchmark our own Great Minds agents
**Tech:** Cloudflare Workers + D1 for storage + simple React dashboard
**Market size:** Every company deploying AI agents (thousands, growing exponentially)
**Session scope:** API endpoints + minimal dashboard + docs

## Idea 2: **WP Autopilot** — AI Site Builder Inside WordPress

Squarespace and Wix are eating WordPress's lunch because they're *easier*. But WordPress has the ecosystem. The missing piece: an AI that actually understands WordPress.

**WP Autopilot** is a chat interface in wp-admin. "Create a contact page with a form that emails me." It generates the page, installs Contact Form 7 if needed, configures it, and shows you the preview. "Add testimonials to the homepage" — it picks a block pattern, inserts it, pre-fills with placeholder content.

Not just content generation — *site building*. It knows the theme, the plugins, the existing structure.

**User:** Small business owners, freelance web designers
**Distribution:** WordPress.org (45% of the web)
**Dogfood:** We use it to spin up client sites faster
**Tech:** PHP plugin + Claude API + WP REST API for mutations
**Market size:** WordPress powers 810M+ websites — even 0.01% adoption is massive
**Session scope:** Single plugin, scoped to page/post creation + basic plugin management

## Idea 3: **One-Click MCP** — Instant MCP Server Generator

Model Context Protocol is the future of tool use, but setting up MCP servers is friction-heavy. Clone a repo, configure JSON, restart Claude Desktop, debug connection issues.

**One-Click MCP** is a CLI + web dashboard. `npx one-click-mcp init` scaffolds a server. `npx one-click-mcp add-tool "fetch weather for a city"` uses AI to generate the tool implementation. `npx one-click-mcp deploy` pushes to Cloudflare Workers and outputs the config to paste into Claude Desktop.

The AI writes the tool code. You just describe what you want.

**User:** Developers using Claude, AI tinkerers
**Distribution:** npm registry + Claude/Anthropic community channels
**Dogfood:** We use it to rapidly prototype new MCP tools
**Tech:** CLI (Node) + Cloudflare Workers templates + Claude for code generation
**Market size:** Every Claude user who wants custom tools (high growth segment)
**Session scope:** CLI package + 3-4 templates + deploy script
